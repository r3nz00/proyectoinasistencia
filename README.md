Sistema de Gestión de Inasistencias de Docentes
De qué se trata:
 El proyecto consiste en una aplicación para registrar y publicar las inasistencias de los docentes, enfocándose principalmente en licencias médicas como justificativo.
 Su propósito es hacer accesible la información de las ausencias, permitiendo que el personal administrativo gestione los datos de manera eficiente y que el resto del personal y los alumnos puedan conocer en tiempo real las licencias activas.


Qué realiza el proyecto
Este proyecto permite registrar y publicar las inasistencias de los docentes, especialmente las licencias médicas. Con él, el personal administrativo puede gestionar toda la información de manera organizada y sencilla.
Además, la aplicación muestra en tiempo real qué docentes están ausentes, de manera que tanto los demás docentes como los alumnos puedan estar informados de las licencias activas. También permite consultar historiales de ausencias, filtrar por fecha, docente o tipo de licencia, y facilita el seguimiento de las inasistencias de forma clara y rápida.
En resumen, ayuda a mantener toda la información de ausencias centralizada, accesible y actualizada, haciendo más eficiente la gestión administrativa.

Herramientas y programas:
XAMPP: Para tener un servidor local con MySQL y phpMyAdmin, donde se guarda la información de las inasistencias.

NetBeans: Para abrir, editar y ejecutar la aplicación en Java.

Figma: Para diseñar la interfaz y los prototipos de la aplicación.

draw.io: Para crear diagramas de flujo y de la base de datos.

GitHub: Para guardar el código, controlar versiones y mantener la documentación del proyecto

El proyecto utiliza los siguientes lenguajes y tecnologías:
Java: Para desarrollar toda la aplicación, incluyendo la lógica del sistema (backend) y la interfaz de usuario (frontend).

SQL (MySQL): Para gestionar la base de datos donde se almacenan las inasistencias y otra información del sistema
